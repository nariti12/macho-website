# マチョクリッカー ブラウザ監査

最終確認: 2026-07-17

## 比較した画面構造

Cookie Clicker公式版のHTMLとゲーム画面資料を確認し、役割を次のように分解しました。

| 領域 | Cookie Clicker | マチョクリッカー |
| --- | --- | --- |
| 主操作 | 大きいCookie、周囲のCursor、所持数とCPS | 大きいマチョ田、周囲のダンベル、筋肉ポイントと毎秒生産 |
| 生産世界 | 建物ごとに独立した行と背景 | 設備ごとに独立した背景、所有アイコン、設備レベル |
| ショップ | Upgradeアイコン、購入可能状態、建物一覧 | Upgradeアイコン、次の解放、購入可能状態、設備一覧 |
| 補助情報 | News、Stats、Options、Legacy | Macho News、統計、表示設定、遺産、日課、実績 |

公式ページ: https://orteil.dashnet.org/cookieclicker/

## スクリーンショット監査

Playwrightで毎回次のサイズを撮影し、ページ全体の横方向オーバーフローも数値で検査します。

- PC 1440 x 900
- PC 1920 x 1080
- スマホ 390 x 844

初回監査では1440px時の中央設備列が細く、設備名と所有数が詰まる問題を確認しました。PCの3カラム比率を `1.3fr / 0.7fr / 360px` へ調整し、中央列の最低幅を確保しています。

スマホでは上部4指標が1行に詰まっていました。390pxでは2列、640px以上では4列へ切り替えるよう修正しました。実績通知も上部タイトルを隠さないよう、スマホでは画面下部へ移動しています。

## 操作・継続テスト

`npm run test:macho-clicker` で次を確認します。

- PC: クリック、ダンベル購入、効果音ON/OFF、軽量モード、5秒オートセーブ、再読み込み後の復元
- スマホ: クリック/ショップ/ジム/統計タブ、ショップスクロール、手動保存
- PC/スマホ: 30分離脱相当からの復帰後も生産・オートセーブ・画面幅が正常
- ゴールデンプロテイン: 5分後の出現、クリック、Lucky報酬
- 音声: click、buy、blocked、achievement、golden-spawn、golden-collect の6ファイルが配信可能

長時間の能動進行は決定論的な60分シミュレーションで、30分離脱からの復帰は実ブラウザと実セーブデータで検証します。テスト失敗時はスクリーンショットとtraceを `test-results/` に出力し、成果物はGit管理しません。

## 判定

- 1440px、1920px、390pxで横方向の画面切れなし
- 主操作、設備世界、ショップの3領域を初見で区別可能
- PCとスマホで購入と保存復元が継続動作
- 画像の縦横比崩れや設備名との重なりなし
- 長時間相当テスト後もポイントと最終保存時刻が前進

今後レイアウト、設備画像、タイマー、保存形式を変更した場合は、lint/buildだけでなくこのブラウザテストも実行します。
