# マチョクリッカー Cookie Clicker 仕様寄せ TODO

## 目的

マチョクリッカーを「ただのクリック画面」ではなく、Cookie Clicker のように長く触りたくなる放置・増殖ゲームへ近づける。

現状は建物価格カーブや一部の数値は Cookie Clicker に寄せているが、以下が不足している。

- 画面を見た瞬間にゲームだと分かるビジュアル密度
- クリック対象・設備・ショップ・アップグレードの視認性
- 序盤から中盤にかけての難易度と購入テンポ
- アップグレード、実績、統計、イベントなどの継続要素
- Cookie Clicker 特有の「次の解放が気になる」情報設計

## 参考にした Cookie Clicker 仕様

- 初期クリックは 1 cookie から始まり、建物購入で自動生産が増える。
- 建物価格は同じ建物を買うたびに `1.15` 倍で上昇する。
- Big Cookie はクリック時に縮小し、粒子と獲得量表示が出る。
- Cursor の所有数は Big Cookie の周囲に表示される。
- Golden Cookie はランダム位置に出現し、一時的な生産増加などの効果を与える。
- Ascension は累計生産量に対する三乗根カーブで、1 prestige level あたり +1% CpS。
- 実績、Milk、Kitten upgrade、季節イベント、Sugar Lump、ミニゲーム、Wrinkler など、長期プレイ用の層が多い。

## 現状で反映済み

- 建物価格上昇率は `1.15`。
- 建物の基本価格と毎秒生産量は、Cookie Clicker の主要建物カーブに近い並び。
- Javascript Console 相当として `筋肉コンソール` を追加済み。
- Ascension 相当の `仕上げ直し` は `1 trillion` 相当から開始し、三乗根カーブで計算。
- Golden Cookie 相当の `ゴールデンプロテイン` は `5〜15分` 出現、`13秒` 寿命。
- Frenzy は `77秒 x7`、Click Frenzy は `13秒 x777` に寄せている。
- クリック時に毎秒生産量が乗る強化は強すぎたため、`0.1%` に下げて解放条件を遅らせている。
- セーブ、インポート、エクスポート、ローカル保存、ランキング送信は実装済み。

## 現状の問題

- 見た目が Cookie Clicker より弱く、ゲーム専用画面というより Web UI に見える。
- 中央クリック領域の背景がまだ情報として弱く、マチョ田がどこにいるのか直感的に伝わりにくい。
- 設備エリアの背景やアイコンが、各建物ごとの世界観としてまだ弱い。
- ショップ、アップグレード、統計がそれぞれ「ゲーム内パネル」ではなく通常カードUIに見える。
- 序盤の手動強化が単純で、プレイヤーが短時間で進みすぎたように感じやすい。
- Cookie Clicker と比べてアップグレード数、実績数、長期要素が少なく、足止めと目標が不足している。

3. 序盤の難易度を Cookie Clicker と実測比較して調整する。
4. 実績を増やし、実績数による恒常ボーナスを追加する。
5. Golden Protein の抽選と演出を強化する。

## 判断メモ

- 「マチョクリッカーの方が簡単に感じる」は妥当。特に序盤はアップグレード数と足止めが少なく、クリック強化も単純なため進行が軽く見える。
- Cookie Clicker と完全同等にするには、数値だけでなく、解放待ち、実績、長期報酬、見た目の変化、統計の充実が必要。
- 次の実装修正では、まず UI と序盤難易度を優先する。長期要素を先に増やすと、今の見づらさが残ったまま複雑になる。
