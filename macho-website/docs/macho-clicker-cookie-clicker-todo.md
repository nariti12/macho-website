# マチョクリッカー Cookie Clicker 超え TODO

## 目的

マチョクリッカーを「ただのクリック画面」ではなく、Cookie Clicker のように長く触りたくなる放置・増殖ゲームへ引き上げる。

最終目標は Cookie Clicker の主要仕様を押さえたうえで、マチョ田の世界観、筋トレ要素、見た目の変化、遊びやすさで独自性を出し、単なる模倣ではなく「マチョ田のサイトに置く意味があるゲーム」にすること。

現状は建物価格カーブや一部の数値は Cookie Clicker に寄せているが、以下が不足している。

- 画面を見た瞬間にゲームだと分かるビジュアル密度
- オレンジ一色ではなく、施設ごとに色と世界観が変わる楽しさ
- クリック対象・設備・ショップ・アップグレードの視認性
- 序盤から中盤にかけての難易度と購入テンポ
- アップグレード、実績、統計、イベントなどの継続要素
- Cookie Clicker 特有の「次の解放が気になる」情報設計

## 完成判定

以下を満たせない場合は「Cookie Clicker と同等/それ以上」とは判断しない。

- 画面を開いた瞬間に、通常のWebページではなくゲーム画面だと分かる。
- クリック、購入、アップグレード、実績解除、ゴールデン系イベントの全てに気持ちよい反応がある。
- 序盤30分の進行テンポが Cookie Clicker と比べて極端に簡単でも単調でもない。
- 1日、1週間、1か月単位で戻ってきたくなる長期目標がある。
- スマホでも「クリックする」「買う」「次の目標を見る」が迷わずできる。
- 見た目、音、アニメーション、UI、数値バランスを実プレイで確認してからリリースする。

## 参考にした Cookie Clicker 仕様

- 初期クリックは 1 cookie から始まり、建物購入で自動生産が増える。
- 建物価格は同じ建物を買うたびに `1.15` 倍で上昇する。
- Big Cookie はクリック時に縮小し、粒子と獲得量表示が出る。
- Cursor の所有数は Big Cookie の周囲に表示される。
- Golden Cookie はランダム位置に出現し、一時的な生産増加などの効果を与える。
- Ascension は累計生産量に対する三乗根カーブで、1 prestige level あたり +1% CpS。
- 実績、Milk、Kitten upgrade、季節イベント、Sugar Lump、ミニゲーム、Wrinkler など、長期プレイ用の層が多い。
- 実績は通常実績だけでも大量にあり、単なる称号ではなく Milk 経由でゲーム進行にも関わる。
- Sugar Lump は長期育成リソースとして建物レベルやミニゲーム解放に関わる。
- Ascension 後は Heavenly Chips / Heavenly Upgrades によって周回プレイの目標が増える。
- Garden / Stock Market / Pantheon / Grimoire のようなミニゲームが、中盤以降の遊びの幅を広げている。

## 現状で反映済み

- 建物価格上昇率は `1.15`。
- 建物の基本価格と毎秒生産量は、Cookie Clicker の主要建物カーブに近い並び。
- Javascript Console 相当として `筋肉コンソール` を追加済み。
- Ascension 相当の `仕上げ直し` は `1 trillion` 相当から開始し、三乗根カーブで計算。
- Golden Cookie 相当の `ゴールデンプロテイン` は `5〜15分` 出現、`13秒` 寿命。
- Frenzy は `77秒 x7`、Click Frenzy は `13秒 x777` に寄せている。
- クリック時に毎秒生産量が乗る強化は強すぎたため、`0.1%` に下げて解放条件を遅らせている。
- セーブ、インポート、エクスポート、ローカル保存、ランキング送信は実装済み。

## 現状の問題

- 見た目が Cookie Clicker より弱く、ゲーム専用画面というより Web UI に見える。
- 中央クリック領域の背景がまだ情報として弱く、マチョ田がどこにいるのか直感的に伝わりにくい。
- 設備エリアの背景やアイコンが、各建物ごとの世界観としてまだ弱い。
- ショップ、アップグレード、統計がそれぞれ「ゲーム内パネル」ではなく通常カードUIに見える。
- 序盤の手動強化が単純で、プレイヤーが短時間で進みすぎたように感じやすい。
- Cookie Clicker と比べてアップグレード数、実績数、長期要素が少なく、足止めと目標が不足している。
- 各設備の背景、アイコン、演出の差が弱く、設備を増やしたときの嬉しさが薄い。
- スマホではゲーム全体の情報量を扱いきれておらず、操作対象が分かりづらくなる場面がある。
- 実装後の難易度比較とプレイテスト基準が不足しているため、「簡単すぎる/難しすぎる」の判断が感覚に寄りやすい。

## 重要メモ: UIは全面再設計が必要

現状のチェック済み項目は、あくまで「機能として入った」状態。ユーザーが求めている Cookie Clicker 級、またはそれ以上のゲーム品質としては未達。

次の実装では、単にCSSを足すのではなく、以下を前提にUIを作り直す。

- WebページのカードUIではなく、ゲーム画面としてデザインする。
- すべての主要UIに専用アート、専用背景、専用アイコン、専用アニメーションを用意する。
- マチョ田の本体キャラクター以外は、既存UIを流用せず全面的に見直す。
- オレンジはブランド色/重要アクション/クリック報酬の強調に限定し、背景や施設は多色で設計する。
- 画像生成を使う場合は、用途ごとにサイズ、構図、比率、余白、文字重なりを指定して生成し、実画面で検証する。
- 生成画像が荒い、意味が分からない、比率が合わない、似たり寄ったりの場合は採用しない。
- Cookie Clicker の強みである「画面全体の統一感」「中央クリック対象の存在感」「右ショップの押したくなる質感」「建物エリアの密度」を基準に評価する。

### アートディレクションメモ

- 方向性: レトロな放置ゲームUI、ジム×筋肉、木目/黒鉄/紙ラベル、施設ごとに変わる色と世界観。
- 画面構造: 左はマチョ田を押す主舞台、中央は設備が増殖するジム、右はショップ棚、下は統計/実績/長期育成。
- 避けるもの: 抽象的な背景、ただの白カードUI、文字と画像の重なり、同じ背景/同じアイコンの使い回し、画面全体が同じオレンジに見える配色。

### 配色方針メモ

- 2026年のUI/ゲーム系トレンドとして、単色ミニマルよりも高彩度、レトロ、ドーパミン配色、ソフトな空間表現、季節イベント色が強い。
- ただし読みやすさが最優先。文字と背景は高コントラストを維持する。
- 推奨パレット:
  - 共通UI: 黒鉄、濃紺、チャコール、紙ベージュ
  - クリック領域: ジムらしい暖色照明 + 床/鉄/鏡のニュートラル
  - 低位施設: ブルーグレー、レザー茶、プロテイン白、食堂イエロー、ベンチ赤
  - 中位施設: トレーナー緑、巨大ジム青、サプリ配送パープル、栄養ラボエメラルド
  - 高位施設: ポータル紫、時空インディゴ、反重力シアン、プリズム虹、コンソールマゼンタ、最終施設ゴールド
- オレンジはブランド色として残すが、全背景をオレンジにしない。
- 各施設の背景画像を作る時は、その施設のメインカラーを1つ、補助カラーを1つ、アクセントカラーを1つ決めてから生成する。

## 判断メモ

- 「マチョクリッカーの方が簡単に感じる」は妥当。特に序盤はアップグレード数と足止めが少なく、クリック強化も単純なため進行が軽く見える。
- Cookie Clicker と完全同等にするには、数値だけでなく、解放待ち、実績、長期報酬、見た目の変化、統計の充実が必要。
- 次の実装修正では、まず UI と序盤難易度を優先する。長期要素を先に増やすと、今の見づらさが残ったまま複雑になる。
- 「TODOを全部実装すれば同じクオリティー」と考えるのではなく、実プレイ、比較、再調整まで含めて品質を上げる。
- Cookie Clicker を超えるには、筋トレという独自テーマを数値だけでなく、見た目、成長、称号、イベントへ反映する必要がある。
