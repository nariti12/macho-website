# マチョクリッカー UI確認チェックリスト

## 目的

マチョクリッカーを修正するたびに、通常のWebページではなく「ゲーム画面」として成立しているかを確認するためのチェックリスト。

## 確認サイズ

- PC: 1920px 幅
- PC: 1440px 幅
- タブレット: 768px〜1279px 幅
- スマホ: 390px 幅

## 画面全体

- 画面を開いた瞬間にゲームだと分かるか。
- 左のクリック領域、中央のジム設備、右のショップの役割が迷わず分かるか。
- 余白がスカスカに見えないか。
- オレンジに偏りすぎず、施設ごとの色と世界観が変わっているか。
- オレンジはCTA、獲得量、重要報酬のアクセントとして効いているか。
- 文字背景は高コントラストで、派手な配色でも読みやすいか。
- 通常のTailwindカードや白いWebカードに見える箇所が残っていないか。

## クリック領域

- マチョ田が画面の主役として見えるか。
- 背景がジム内装として伝わるか。
- ダンベル、粒子、数字、背景演出がキャラクターに被りすぎていないか。
- クリック時に縮小、反発、獲得量表示、音が分かるか。
- スマホでも「ここを押すゲーム」と直感的に分かるか。

## ジム設備

- 各設備の背景とアイコンが区別できるか。
- 高位施設ほどスケール感、発光、演出が強くなっているか。
- 専属トレーナー以降の背景画像が、ベンチプレス軍団までと比べて急に抽象的・低品質になっていないか。
- 高位施設の背景に、施設名から連想できる具体物、人物、器具、空間が入っているか。
- 施設を購入した時に、対象行へ追加されたことが分かるか。
- 所有数が増えても右に切れて見えなくならないか。
- 文字とアイコン、背景が重なって読みにくくなっていないか。

## ショップ

- 購入可能、購入不可、未解放、取得済みの状態が一目で分かるか。
- 購入済みアップグレードがメインショップを圧迫していないか。
- 取得済みアップグレードのアイコンが並ぶ場合、何の効果か分かるか。
- Cookie Clicker と比べて、購入済みアップグレードの表示量が多すぎないか。
- 購入可能な行は押せそうに見えるか。
- 購入不可でも不足分が分かるか。
- ホバー詳細が対象の近くに出るか。
- スクロール不能や、ツールチップが画面外に飛ぶ問題がないか。

## 統計・長期要素

- 実績、プロテインシェイク、筋肉結晶、仕上げ直しがゲーム内パネルに見えるか。
- 次の目標が分かるか。
- 数字が読みやすいか。
- 英語表記、日本語表記、全桁表示が破綻しないか。

## スマホ

- 固定タブで「クリック」「ジム」「ショップ」「統計」を迷わず切り替えられるか。
- ショップ内でスクロールできるか。
- クリック領域が小さすぎないか。
- 重要な数字が折り返して読みにくくなっていないか。

## 軽量モード

- 背景、粒子、常時アニメーションが止まるか。
- クリック、購入、実績解除など最低限の操作感は残るか。
- スマホで重くなりすぎないか。
