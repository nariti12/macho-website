# 集客・アフィリエイト運用

少ない作業時間でも改善を継続できるよう、計測、記事、X、商品ページの運用を1つにまとめます。実装仕様は `docs/site-spec.md` を参照してください。

## 最初に見る指標

月間ユーザー数が少ない間は、売上だけで判断せず次の順で確認します。

1. Google Search Consoleの表示回数と検索クエリ
2. GA4の自然検索ユーザー数と閲覧ページ
3. `affiliate_click` の回数とクリック率
4. Amazon・楽天・iHerbごとのクリック数
5. 各アフィリエイト管理画面の注文・報酬

クリック率は、商品ページの `affiliate_click` 数をそのページの表示回数で割って確認します。流入が少ない段階では、日次の上下ではなく28日単位で比較します。

## affiliate_click

外部ECボタンを押すと、GA4へ次のイベントを送信します。

| 項目 | 内容 |
| --- | --- |
| `affiliate_merchant` | `amazon` / `rakuten` / `iherb` |
| `item_name` | 表示中の商品名 |
| `item_rank` | ページ内の順位 |
| `page_path` | クリックしたページ |
| `placement` | プロテイン、シューズ、ギアなどの配置 |
| `link_url` | 遷移先URL |

GA4の管理画面で `affiliate_merchant`、`item_name`、`item_rank`、`placement` をイベントスコープのカスタムディメンションとして登録します。成果につながる主要操作として扱う場合は、`affiliate_click` をキーイベントに設定します。実際の購入は各アフィリエイト管理画面で確認します。

## 週2〜3時間の運用

### 1. 数字確認: 20分

- Search Consoleで表示回数が増えたクエリを確認する
- GA4で商品ページごとの閲覧数と `affiliate_click` を確認する
- クリックがあるのに売上がない商品は、価格、リンク先、説明の一致を確認する

### 2. 既存ページ改善: 40分

- 表示回数が多いのにクリック率が低いページを1つ選ぶ
- 冒頭文、商品画像、参考価格、注意点、リンク先を確認する
- 実際に使った商品は、使用期間、良かった点、合わなかった点を追記する
- 未使用の商品を使用済みのように記載しない

### 3. BlogまたはFAQ追加: 60分

- 自分のトレーニングで実際に起きたことを1テーマだけ書く
- 結論、実施内容、結果、失敗点、次に変えることの順にまとめる
- 関連する計算機、FAQ、おすすめ商品へ自然に内部リンクする
- 商品を紹介する場合は、誰に向くかと注意点を必ず書く

### 4. X投稿: 30分

- Blogの要点を1投稿に短くまとめ、詳細ページへ誘導する
- 商品リンクだけを連投せず、経験や比較理由を本文に含める
- 同じ内容でも結論、失敗談、数値の3パターンを試す
- サイトへのリンクには流入元を判別できるUTMを付ける

例:

```text
https://www.machoda.com/blog/ARTICLE_ID?utm_source=x&utm_medium=social&utm_campaign=training_log
```

## 月1回の見直し

- 28日間とその前の28日間で、検索表示回数、自然検索ユーザー、アフィリエイトクリックを比較する
- クリックされない商品は順位を固定したまま放置せず、画像、価格、説明、リンク先を再確認する
- 売上が発生したページは、関連FAQや実体験記事を追加して入口を増やす
- 古い価格や販売終了商品を確認する
- 記事テーマはSearch Consoleで実際に表示された検索語を優先する

## コンテンツ方針

- 一般論を大量に書くより、マチョ田が実際に試した内容を優先する
- おすすめ商品は数を増やしすぎず、選定理由と欠点を明確にする
- 医療・栄養に関する断定は避け、必要に応じて公的機関や一次情報を参照する
- 広告リンクの有無で順位や評価を変えない
- ゲームは再訪のきっかけ、FAQ・計算機・商品ページは検索流入と課題解決の入口として役割を分ける
